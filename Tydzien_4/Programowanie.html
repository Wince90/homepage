24.03.2020 Dzień 5 - Podstawowe działanie strony, javascriptu i termianala

Działanie strony

Dowiedziałem się jak działa ruch na stronie. w pierwszej kolejności wpisywany jest adres strony np. onet.pl. następnie przeglądarka za pomocą DNS’u tłumaczy wpisaną frazę na adres. Po znalezieniu serwera odpytuje go o index (w zależności co jest plikiem indeksu na serwerze. Następnie serwer wysyła do przeglądarki zawartość html. Po czym przeglądarka wczytuje html i szuka dodatkowych zasobów .css, .js .jpg … dla każdego z zasobów wykonuje zapytanie. Przeglądarka parsuje (dokonywanie analizy składniowej ciągu znaków w celu ustalenia jej struktury by wiki) i renderuje stronę.
Kody protokołu HTTPs
100 > Be patient, not ready.
200 > Its ok.
300 > Wrong place. Let i show you the way.
400 > 404 ^-(o_o)-^ Nothing in here.
500 > Ups we got a problem.
Istotną kwestią przy tworzeniu wszelkich linków lokalnie jest nie zamieszczanie ścieżki roota np : img src”img/wtf.svg” alt=”Logo WTF: Co ten front end” - ścieżka względna
Jeżeli już umieszczamy projekt na serwerze, należy dodawać ścieżkę roota np. img src” style color: red;>/img/wtf.svg” alt=”Logo WTF: Co ten front end”/> - ścieżka bezwzględna.
Różnica pojawia się wtedy gdy pojawiają się jakiekolwiek podstrony na serwerze.

Terminal

Podstawowe komendy dotyczące poruszanie się w terminalu w systemie Windows. cd nazwa-katalogu - przejście do katalogu o podanej nazwie + Tab - automatyczne znajdywanie istniejącej ścieżki po wpisaniu pierwszych liter. cd .. - przejście o jedną pozycję wyżej, cd ..\.. - 2 pozycje. code . - uruchomienie w danej ścieżce vcs. dir - wypisanie zawartości katalogu. mkdir nazwa-katalogu - stworzenie katalogu o podanej nazwie.

26.03.2020 Dzień 7 - Jest Git ;)

GIT

Nauczyłem się podstawowych koment i pracy z repozytorium.

    Podstawowe:
  • Komenda do nadania imienia i nazwiska:
    git config --global user.name “Imie Nazwisko”
  • nadanie e maila:
    git config --global user.email “email@email.com”
  • Informacja zwrotna imię nazwisko :
    git config --get user.name
  • analogicznie e mail :
    git config --get user.email
    Projekt:
  • git init (dodaje nowy plik git do śledzenia)
  • git status (sprawdza aktualny status plików ;P
  • git add . (dodaje wszystkie pliki do aktualizacji bez kropki pojedynczy plik)
  • git commit -m “nazwa zmian” (dodaj komentarz do zmian)
  • gitk (status wszystkich komentarzy)
Flow działania: git add . / git commit -m “opis zmian” / git push

27.03.2020 Dzień 8 Box model

Część I

Dziś zgłębiłam tajniki “Snake mode on” pdk ;) display : block; - element wyświetla się na całą szerokość rodzica.
przykład_______________________________________________________
display : inline; - element zajmuję tylko potrzebne miejsce np.“tutaj”. W całym tekście.
Wysokości i szerokości ( width and height) działają przy użyciu display : block i display : inline-block.
Prostą wersją responsywności strony jest połaczenie 2 wartości. width : 100px; - ustawia na “twardo” wartość szerokości.
width : 100%; + max-width : 700 px - ustala szerokość na 700 px ale po zmniejszeniu szerokości “zawija” zawartość (taka podstawa responsywności).
Wartości : vw - szerokość w procentach ale bez dziedziczenia względem rodzica . dostosowuje się do wymiarów strony automatycznie. np width : 50vw;

Część II


div - nie ma znaczenia semantycznego. Używany jest w momencie braku odpowiedniego znacznika.
box-sizing : content-box - powierzchnia na zawartość się nie zmniejsza, każde dodanie zwiększa wielkość całego elementu (pudełka).
border-box - zmienia się powierzchnia na zawartość, ale zostają wymiary wcześniej określone.
position: relative - Tak pozycjonowany element staje się kontenerem dla wszystkich innych, które są absolutnie w nim pozycjonowane.
